# Controlling the Dangbei projector via MQTT (ESP32 project)

Проект эмуляции пульта для проектора Dangbei (тестировалось на Mars Pro) на плате ESP32-arduino (тестировалось на NodeMSU-32s). Цель -- управление проектором из умного дома основанном из WirenBoard (power/volume/навигация). 

Попутно в пульте обнаружилась функция поиска пульта (пищит нарастающим писком) и можно вытащить % заряда пульта, но для этого надо держать соединение между пультом и ESP, а нормально реализовать это не получилось, поэтому я отказался от этой идеи, возможно у вас получится. 

Скетч для ESP32, который делает “мост” между:
- **проектором Dangbei** (BLE HID server)
- **MQTT** (управление и публикация статусов)
~~- **физическим пультом** (BLE HID client)~~

## Как это работает (архитектура)

Есть **два независимых BLE-соединения**:

1. **Проектор <-> ESP**: ESP выступает BLE **HID сервером** (`1812`), отправляет HID notify для consumer + keyboard.
2. ~~**ESP <-> пульт**: ESP выступает BLE **клиентом** к физическому пульту (подписка на notify `2A4D`), получает payload-и и пробрасывает их в HID сервер проектора.~~

## MQTT

Разрабатывался для моего умного дома, ядром которого является Wirenbord, отсюда формат сообщений описанный ниже.

Общий шаблон:
`/devices/<DEVICE_ID>/controls/<name>/on` — команды (payload зависит от типа контрола)

### Команды


| Имя                                     | Тип        | Действие                                                                                                |
| --------------------------------------- | ---------- | ------------------------------------------------------------------------------------------------------- |
| `power`                                 | pushbutton | если проектор подключён — отправляет `sendKeyboard(0x66)`, иначе — запускает wake FSM (`startWakeUp()`) |
| `volume_up`                             | pushbutton | отправка consumer `0x00E9`                                                                              |
| `volume_down`                           | pushbutton | отправка consumer `0x00EA`                                                                              |
| `home` / `menu`                         | pushbutton | consumer `0x0223` / `0x0040`                                                                            |
| `up/down/left/right/ok/back/quick_menu` | pushbutton | keyboard `0x52/0x51/0x50/0x4F/0x28/0x29/0x3F`                                                           |
| `find_pult`                             | pushbutton | переключает зуммер на физическом пульте (FFF0/FFF2/FFF3)                                                |
| `remote_pult_on`                        | pushbutton | включает BLE-проброс с физического пульта (появляется в логике как `gPult.enabled`)                     |
| `volume`                                | range      | значение приходит в payload: внутри собирается `volume_set_<N>` и вызывается `setVolume(N)` (0..20)     |


### Публикация retained-метаданных

После подключения к брокеру выполняется `publishMeta()`, публикуются retained-поля `meta/type/meta/title/meta/order` и топики `.../on`.

Сенсоры/статусы:

- Батарея пульта: топик  
`/devices/<DEVICE_ID>/controls/remote_pult_battery`  
  - метаданные в `.../meta/`* (тип `text`, order `13`). Значение: `--` или процент.
- Подключённость проектора: топик  
`/devices/<DEVICE_ID>/controls/connected`  
  - метаданные в `.../meta/*` (тип `text`, order `17`). Значение: `1/0`.
- Подключённость пульта: топик  
`/devices/<DEVICE_ID>/controls/pult_connected`  
  - метаданные в `.../meta/*` (тип `text`, order `18`). Значение: `1/0`.

## Настройки (константы)

Перед сборкой нужно в скрипте указать настройки подключения. Параметры находятся в `pult.ino`:

- `WIFI_SSID`, `WIFI_PASSWORD` (wifi твоей сети)
- `MQTT_HOST`, `MQTT_PORT`, `MQTT_USER`, `MQTT_PASS` (mqtt брокер умного дома)
- `DEVICE_ID` (встроен в MQTT topics, у меня используется как имя устройства в умном доме)
- `PAIR_TIMEOUT_MS`, `WAKE_DURATION_MS` (здесь можно оставить как есть)
- ~~`FIND_PULT_REMOTE_ADDRESS` (MAC адрес физического пульта)~~
- ~~`PULT_RECONNECT_BASE_MS`, `PULT_RECONNECT_MAX_MS`  (здесь можно оставить как есть)~~

## Сборка/загрузка

1. Установи ESP32 board package (Arduino-ESP32) под ESP32. (у меня используется плата NodeMSU-32s)
2. Добавь зависимости:
  - `NimBLE-Arduino` (нужен для `NimBLE*`)
  - `PubSubClient` (MQTT)
3. В Arduino IDE выбери плату ESP32 и загрузочный порт.
4. Задай Wi-Fi/MQTT параметры в `pult.ino`, если ещё не сделал.
5. Залей прошивку.
